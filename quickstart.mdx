---

<Steps>

<Step title="Add the dependency">

Add MockK to your test dependencies. Replace `{mockkVersion}` with the [latest version from Maven Central](https://search.maven.org/artifact/io.mockk/mockk).

<CodeGroup>

```kotlin build.gradle.kts
dependencies {
    testImplementation("io.mockk:mockk:{mockkVersion}")
}
```

```groovy build.gradle
dependencies {
    testImplementation "io.mockk:mockk:{mockkVersion}"
}
```

```xml pom.xml
<dependency>
    <groupId>io.mockk</groupId>
    <artifactId>mockk-jvm</artifactId>
    <version>{mockkVersion}</version>
    <scope>test</scope>
</dependency>
```

</CodeGroup>

<Note>
For Android projects, use the `mockk-android` artifact instead. See [Android support](/android).
</Note>

</Step>

<Step title="Write your first mock">

Create a mock with `mockk<T>()`, define its behavior with `every`, call the code under test, then verify with `verify`.

```kotlin
import io.mockk.confirmVerified
import io.mockk.every
import io.mockk.mockk
import io.mockk.verify

val car = mockk<Car>()

every { car.drive(Direction.NORTH) } returns Outcome.OK

car.drive(Direction.NORTH) // returns Outcome.OK

verify { car.drive(Direction.NORTH) }

confirmVerified(car)
```

By default, mocks are strict: calling a method that has no stub configured throws an exception. If you want a mock that returns zero-values for everything, use `mockk<Car>(relaxed = true)` instead.

</Step>

<Step title="Run your tests">

Run your test suite the same way you normally would:

```bash
./gradlew test
```

MockK hooks into the standard test lifecycle. No additional test runner configuration is required for JVM projects.

</Step>

</Steps>

## BDD-style aliases

If your team uses Behavior-Driven Development, MockK provides a separate artifact with `given`/`then` aliases for the core stubbing and verification functions.

Add the BDD dependency alongside the main one:

```kotlin build.gradle.kts
dependencies {
    testImplementation("io.mockk:mockk:{mockkVersion}")
    testImplementation("io.mockk:mockk-bdd:{mockkVersion}")
}
```

The alias mapping:

| Standard MockK | BDD alias |
|---|---|
| `every { ... }` | `given { ... }` |
| `coEvery { ... }` | `coGiven { ... }` |
| `verify { ... }` | `then { ... }` |
| `coVerify { ... }` | `coThen { ... }` |
